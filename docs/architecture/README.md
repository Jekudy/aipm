## суть проекта в одну строчку

мы строим не генератор документов, а ai-систему, которая помогает продакту двигать продуктовую работу по зрелости: от сигнала к opportunity, от opportunity к решениям, от решения к prd, от backlog-а к квартальному плану.

## главный принцип

плагин не должен начинать с генерации документа. он должен начинать с диагностики состояния работы:

1. что принес пользователь?
2. какая это сущность?
3. есть ли похожая сущность в backlog?
4. на каком maturity level она сейчас?
5. какой canonical flow применим?
6. чего не хватает для следующего meaningful decision?
7. что может сделать ai?
8. что должен подтвердить pm?

только после этого плагин генерирует, ревьюит, обновляет документы или двигает item дальше.
